What are the best practices associated with using classes vs. ids?

We would use the CLASS attribute to specify a style for a mixed group of element tags giving them all styles they will then have in common. It is possible to implement multiple classes in a single class tag (examples in picture bellow). It's important to rememeber when you insert your class name to a class attribute that you don't add spaces as it would be read as a different class, making the inteded one void showing the default style results.


We would use the ID attribute when we have a single element on our page that we want to hookup to a link or styled in a specific way seperate from the class styles. It's considered good practise to not use the same ID tag for multiple different element tags on the same html document as it's supposed to be unique to a specific element, and you would use the class attribute for the purpose of styling more than a single element. Id tags also have a higher specificity/importance over the class attribute. For example, if you were to have an ID and class attribute both set to change the font color, then the ID tag by default of the ID style will show in the result again due to higher specificity, (examples pictured).
